# 00. Getting Started

Welcome to **ABAPSapEcc**, a mini documentation project intended to help developers and system integrators understand the SAP ECC landscape and ABAP development workflow.

---

# SAP System Landscape

A typical SAP ECC environment is separated into multiple systems to ensure development can be performed safely before reaching production.

![SAP System Landscape]https://i.imgur.com/AqmTeN4.png

## Development (DEV)

The Development system is where developers write and modify ABAP programs, reports, function modules, SmartForms, and other SAP objects.

### Characteristics

- New developments are created here.
- Existing programs are enhanced or fixed.
- Unit testing is performed by developers.
- Objects are transported to the next environment after completion.

---

## Quality Assurance (QA)

The Quality Assurance system is used for validation before changes are released to production.

### Characteristics

- Functional Testing
- Integration Testing
- User Acceptance Testing (UAT)
- Verification of Transport Requests

Only approved transports should move beyond this stage.

---

## Production (PRD)

The Production system is the live environment used by end users.

### Characteristics

- Contains live business data.
- Used for daily business operations.
- Direct development is generally prohibited.
- Changes are deployed through approved Transport Requests.

# Configuring SAP Logon

Before logging in, ensure your SAP Logon connection has been configured correctly.

![SAP Logon Configuration]https://i.imgur.com/7MApr2j.png

Your SAP Basis or System Administrator should provide the following information:

- Description
- Application Server
- Instance Number (System Number)
- System ID (SID)
- SAP Router (if applicable)

Once these settings are configured, the SAP system will be available from SAP Logon.

---

# Logging In

Open SAP Logon and select the desired SAP system.

Enter your assigned credentials.

![SAP Login Screen]https://i.imgur.com/fxfNCya.png

Required fields:

- Client
- User
- Password
- Language (usually **EN**)

After successful authentication, SAP GUI will open.

> **Note**
>
> If this is your first login, SAP may require you to change your initial password.

---

# SAP Easy Access

After logging in successfully, you will arrive at the **SAP Easy Access** screen.

![SAP Easy Access]https://i.imgur.com/S15YqSV.png

This is the main entry point for navigating SAP ECC.

---

## Transaction Code (T-Code) Command Field

Located at the **top-left corner** of SAP GUI.

This field allows you to execute SAP transactions directly by entering a Transaction Code (T-Code).

Examples:

| Transaction | Description |
|-------------|-------------|
| `SE11` | ABAP Dictionary |
| `SE38` | ABAP Editor |
| `SE80` | Object Navigator |
| `SE37` | Function Builder |
| `SM37` | Background Jobs |
| `ST22` | ABAP Dumps |

Using T-Codes is the fastest way to navigate within SAP.

---

## SAP Menu

Located on the **left navigation panel**.

The SAP Menu organizes transactions by business modules such as:

- Financial Accounting (FI)
- Materials Management (MM)
- Sales and Distribution (SD)
- Production Planning (PP)
- Human Resources (HR)

If you do not know the T-Code of a transaction, you can browse through the SAP Menu.

---

## Favorites

Also located on the **left navigation panel**.

The Favorites menu allows users to bookmark frequently used transactions.

Common examples:

- SE11
- SE38
- SE80
- SE37
- ST22
- SM37
- SM50
- SM66

Using Favorites can significantly improve productivity during daily development.

---

# Tips

- Learn commonly used T-Codes instead of navigating through menus.
- Keep your Favorites organized.
- Always verify which SAP system (DEV, QA, or PRD) you are currently connected to before making any changes.
